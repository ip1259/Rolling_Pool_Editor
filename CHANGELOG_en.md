# Changelog

## Unreleased

### Added

- Added **Export MOD**, which writes the current weight changes directly to a fixed-name `regulation.bin` in a user-selected MOD folder.
- Added a standalone self-contained .NET helper for Nightreign regulation decryption, PARAM editing, repacking, encryption, and output verification.
- Bundled the Regulation Ver. 1.03.5 `Grand Only/regulation.bin` base and the helper with the Windows x64 package; users do not select a custom input regulation or need Smithbox.
- Added localized Export MOD interface text for all 15 supported languages.

### Safety and validation

- Export is refused when no weights have been modified and requires confirmation before overwriting an existing output.
- Only modified rows in the editable Table-ID whitelist can be written. Locked rows, non-editable Tables, and all unrelated regulation content remain unchanged.
- The fixed bundled base is checked by size and SHA-256 and is never overwritten.
- The generated regulation is reopened and compared with the base before it is accepted; failures do not leave a final output file.
- Added Python integration tests, .NET helper tests, and a golden regulation fixture test.
- Confirmed in game that a `regulation.bin` generated by the helper is usable.
- Confirmed that the packaged application starts and exports on a Windows computer without Python, the .NET SDK/runtime, or Smithbox installed, and that its exported `regulation.bin` works in game.
- Added README attribution and acknowledgements for Smithbox and its `Andre.SoulsFormats` library used for Nightreign regulation encryption, decryption, and format handling.

## Release Ver 1.1 — 2026-07-21

### Added

- Added **Auto Configure** to automatically adjust weights across all editable rolling-pool Tables using the currently selected effect tags.
- Added `SpEffectParam` data and `passiveSpEffectId_1` relationships so the editor can determine each effect's `spCategory`.
- Added automatic weight rules based on effect stacking behavior:
  - `spCategory = 10`: the effect can stack with itself. The editable entry with the highest Effect ID in the same filter group receives `+300`; the remaining editable entries are reduced to `1`.
  - `spCategory = 20`: the same effect level cannot stack with itself, but different levels can stack. Entries in the same filter group receive `+300`, `+250`, `+200`, and so on in descending Effect ID order, with a minimum increase of `0`.
  - Other `spCategory` values: matching Effect IDs receive `+300` without extending the adjustment to other levels under the same effect tag.
- Added localized Auto Configure interface text for all 15 supported languages.
- Added a bilingual BBCode description for the Nexus Mods page in `NexusDesc.txt`.

### Changed

- Auto Configure preserves entries whose original final weight is `0` because they are locked.
- Auto Configure preserves entries whose current weight has already been changed by the user.
- Unfiltered editable entries are reduced to `1` instead of `0`.
- The Auto Configure button remains disabled until at least one effect tag is selected.
- Updated the English and Traditional Chinese user manuals with the Auto Configure workflow, stacking-category meanings, and safety notes.
- Cleaned up redundant implementation comments and outdated refactoring notes without changing application behavior.

### Validation

- Verified loading of the expanded game parameter database.
- Verified the rules for locked entries, modified entries, unfiltered entries, and `spCategory` values `10`, `20`, and other values.
- Verified all GUI localization JSON files and required Auto Configure keys.
