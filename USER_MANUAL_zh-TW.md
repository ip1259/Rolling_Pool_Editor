# Rolling Pool Editor 操作手冊

## 1. 程式簡介

Rolling Pool Editor 是用來調整遺物抽獎池權重的開源 Python 圖形化工具。程式會依各項效果的權重即時計算出現機率，並輸出與 Smithbox 相容的 `AttachEffectTableParam.csv`。

輸出的 CSV 必須透過 Smithbox 匯入 `AttachEffectTableParam`，再將修改後的 `regulation.bin` 作為 Mod 使用。完整匯入方式請參閱 [Smithbox 使用說明](SMITHBOX_GUIDE_zh-TW.md)。

## 2. 啟動程式

### Windows x64 打包版

專案提供 Windows x64 打包版本。解壓縮後，執行 `RollingPoolEditor.exe` 即可，不需要另外安裝 Python。

### 從原始碼執行

本專案使用 Python 3.13，並可透過 [uv](https://docs.astral.sh/uv/) 建立及同步虛擬環境。在專案根目錄執行：

```powershell
uv sync
uv run python src/main.py
```

主程式進入點為 `src/main.py`。請從專案根目錄啟動，讓程式能正確讀取 `src/game_data` 內的資料庫。

## 3. 介面概覽

### 系統設定

- **介面與遊戲語系**：同時切換介面文字與遊戲效果名稱。首次啟動時預設為英文。
- **外觀主題模式**：可切換 Dark 與 Light 主題。
- **參數表格 ID**：選擇要編輯的抽獎池。

程式會自動記錄語言、主題及目前勾選的效果標籤。下次啟動時會還原這些設定，不需要手動儲存。Windows 的設定檔位置為：

```text
%APPDATA%\RollingPoolEditor\config.json
```

> [!NOTE]
> 設定檔不會保存權重修改、目前選擇的 Table ID、排序方式或分類的展開／收合狀態。權重修改必須匯出為 CSV 才能保留。

### 效果標籤過濾器

左側過濾器依大分類、中分類及效果標籤排列：

- 按分類前的箭頭可展開或收合內容。
- 勾選中分類會一次選取或取消其下所有效果標籤。
- 勾選個別標籤時，右側只顯示符合條件的效果。
- 未勾選任何標籤時，右側顯示目前 Table 的全部效果。
- 按下「取消所有篩選」可清除全部勾選項目。

### 參數編輯工作區

表格顯示以下資訊：

- **效果 ID**：效果在遊戲參數中的識別碼。
- **效果名稱**：依目前語系顯示的遊戲文本。
- **最終權重**：實際用於機率計算的權重。
- **出現機率**：該效果權重占目前 Table 總權重的比例。
- **修改期望值**：用來輸入新權重、儲存或快速設為 1。

點擊欄位標題可排序；再次點擊同一欄位可切換升冪與降冪。

## 4. Table ID 對照

| Table ID | 用途 |
| --- | --- |
| `100`、`200`、`300` | 商店敘述中的 Ver. 1.02 一般 Relic 抽獎池 |
| `110`、`210`、`310` | Ver. 1.03（含 DLC）一般 Relic 抽獎池 |
| `2000000` | Deep Relic 通用獎池 |
| `2100000` | Deep Relic Ver. 1.02 獎池 |
| `2200000` | Deep Relic Ver. 1.03（含 DLC）獎池 |
| `3000000` | Debuff 池 |

## 5. 修改單一效果的權重

1. 選擇要編輯的 Table ID。
2. 視需要使用效果標籤縮小顯示範圍。
3. 點擊目標效果最右側的權重輸入區。
4. 輸入整數權重。
5. 按 `Enter` 或點擊「✔」儲存。

儲存後，程式會立即重新計算目前 Table 內所有效果的出現機率。

> [!IMPORTANT]
> 程式啟動時，最終權重原本就是 `0` 的效果代表不可能從該獎池抽到，因此會被永久鎖定，無法透過本工具修改。原始權重非 0 的項目則可在本次編輯中手動設為 `0`。

> [!CAUTION]
> 不建議將任何可修改項目的權重設為 `0`。過度移除效果可能使獎池可用項目不足，並可能讓結果不符合遊戲所需條件。請使用非負整數；負數雖可能被輸入，但不代表有效的抽獎池設定。

### 快速設為 1

點擊每列最右側的「1」按鈕，可直接把該效果權重設為 1。這能大幅降低該效果的出現機率，同時避免把它完全移出獎池。原始權重為 0 的鎖定項目無法使用此功能。

## 6. 批次調整功能

### 篩選項權重 +100

先在左側選取一個或多個效果標籤，再按下「篩選項權重 +100」。確認後，程式會將目前畫面中所有可修改項目的權重各增加 100。

若未設定篩選條件，畫面會包含整個 Table，因此此功能會套用至目前 Table 內所有可修改項目。執行前請確認顯示範圍與確認視窗中的項目數量。

### 未修改項設為 1

按下「未修改項設為 1」並確認後，程式會將目前 Table 中符合以下條件的項目設為 1：

- 原始權重大於 0，因此不是鎖定項目。
- 目前權重仍等於原始權重，代表尚未修改。

此功能刻意使用 `1` 而不是 `0`，是為了降低未選中效果的機率，同時確保獎池仍保留足夠數量的可抽取項目。

## 7. 套用到其他 Table

「套用到其他表格」會將目前 Table 中已修改的效果，依效果分類標籤映射至其他可編輯 Table。它不是單純依相同 Effect ID 複製。

例如，「生命力 +1」、「生命力 +2」與「生命力 +3」屬於同一效果標籤，通常 Effect ID 越大代表同類效果越強；「提升生命力」則是另一個標籤，不會被視為同一類。

程式只會處理目前 Table 中原始權重大於 0、而且權重確實已變更的來源項目。其他 Table 中原始權重為 0 的鎖定項目也不會成為套用目標。

映射規則如下：

- **增加權重時**：在目標 Table 的相同效果標籤中，選擇可修改且 Effect ID 最大的項目。例：修改「生命力 +2」後，其他 Table 實際被增加權重的項目可能是「生命力 +3」。
- **降低權重時**：在相同效果標籤中，將所有 Effect ID 小於或等於來源 Effect ID 的可修改項目同步設為來源的新權重。此操作可能一次修改同一目標 Table 內的多個效果。
- 若目標 Table 沒有符合規則的項目，該次修改不會套用至該 Table。
- 套用範圍為本手冊列出的其他可編輯 Table，不包含目前作為來源的 Table。

> [!IMPORTANT]
> 執行後請逐一切換 Table 檢查結果。由於映射依效果標籤、Effect ID 與原始鎖定狀態決定，其他 Table 被修改的效果不一定與來源效果具有相同名稱或 Effect ID。

## 8. 從 CSV 匯入

按下「從 CSV 匯入」並選擇 CSV。匯入檔案必須包含以下欄位：

```text
ID
attachEffectId
chanceWeight
chanceWeight_dlc
```

程式會依 Table ID 與 Effect ID 尋找現有項目，匯入可修改項目的權重，並略過不存在或原始權重為 0 的項目。匯入完成後會顯示更新筆數。

## 9. 重置本次所有修改

按下「重置本次所有修改」後，程式會把整個編輯層重新載入至程式啟動時的初始資料。此操作會清除所有 Table 的手動修改、批次修改與 CSV 匯入結果，不只目前顯示的 Table。

## 10. 驗證與匯出 CSV

1. 完成所有 Table 的權重調整。
2. 按下「檢查並匯出 CSV 參數檔」。
3. 選擇儲存位置與檔名；預設檔名為 `AttachEffectTableParam.csv`。
4. 通過驗證後，程式會輸出 Smithbox 相容的 CSV。

輸出檔保留原始資料列順序與必要欄位，並同時包含 `chanceWeight` 和 `chanceWeight_dlc`。

### Debuff 池驗證

`3000000` 是 Debuff 池。部分配置可能需要同時使用 3 個 Debuff，尤其是效果較強的 Deep Relic，因此此池必須至少保留 3 個最終權重非 0 的項目。

若非 0 項目少於 3 個：

- 介面會顯示警告。
- CSV 將無法通過驗證及匯出。

## 11. Smithbox 與 Mod 載入

匯出後，必須將同一份 CSV 的 `chanceWeight` 與 `chanceWeight_dlc` 分別匯入 Smithbox；Smithbox 一次只能選擇一個欄位，因此需要操作兩次。詳細步驟與 Mod 安全警示請參閱 [Smithbox 使用說明](SMITHBOX_GUIDE_zh-TW.md)。
